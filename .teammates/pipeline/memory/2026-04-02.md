## CI workflow created

Added `.github/workflows/ci.yml` ‚Äî the primary build/test pipeline for Vectra.

**Configuration:**
- Triggers: push to `main`, pull requests to `main`
- `paths-ignore`: docs, .teammates, markdown files (operational metadata doesn't trigger CI)
- Matrix: Node 18, 20 on ubuntu-latest
- Steps: checkout ‚Üí setup-node (with yarn cache) ‚Üí `yarn --frozen-lockfile` ‚Üí `npm run build` ‚Üí `npm run test:mocha`
- Codecov upload on Node 20 only (avoids duplicate reports)
- `fail_ci_if_error: false` on Codecov ‚Äî won't block CI if Codecov is unreachable
- Uses `CODECOV_TOKEN` secret (repo owner must add this)

**Key decisions:**
- Included Node 18 in matrix per request even though `engines` field says `>=20.x`
- Ran build step separately from test (`npm run test:mocha` not `npm run test`) since `test` script re-runs build
- Coverage upload only on one matrix entry to avoid double-counting
- Least-privilege permissions: `contents: read` only

**Repo setup required:** Add `CODECOV_TOKEN` secret in repo settings for coverage uploads.
