### Task 13: Implement `vectra watch` — automatic folder sync (GitHub issue #23)
Added a `FolderWatcher` class and `vectra watch` CLI command that monitors folders for file changes and automatically syncs them into a `LocalDocumentIndex`.

**New library class (`src/FolderWatcher.ts`):**
- `FolderWatcher` class extending `EventEmitter`
- Config: `index`, `paths` (folders/files to watch), `extensions` (filter), `debounceMs`
- `start()` — performs initial full sync, then sets up `fs.watch` watchers with debouncing
- `stop()` — closes watchers, clears pending debounces
- `sync()` — manual full re-scan (detects adds, updates, deletes by mtime comparison)
- Events: `sync(uri, action)`, `error(err, uri)`, `ready`
- Properties: `isRunning`, `trackedFileCount`
- Uses `fs.watch({ recursive: true })` for directories, single-file watchers for files
- Debounces rapid change events to avoid redundant re-indexing

**CLI command:**
- `vectra watch <index> --keys <keys> --uri <path>... / --list <file>` — matches issue's suggested interface
- Options: `--extensions` (filter by ext), `--chunk-size`, `--debounce`
- Auto-detects codec, shows live sync status, graceful Ctrl+C shutdown

**Tests (`src/FolderWatcher.spec.ts`) — 13 tests:**
